# BRAINS.md - 뇌 가동 체계

주인님(필) 명령 기준 최신 운영 규칙입니다.

## 현재 뇌 리스트
1. **1번 뇌 (구글 Gemini CLI / 3 Flash Preview):** 대량 처리/요약/분류
2. **2번 뇌 (안티그래비티 / Gemini 3 Flash):** 수집 보조/간단 실무
3. **3번 뇌 (GPT / 5.3 Codex) [똑뇌]:** 코딩/구현/전략 설계 메인
4. **4번 뇌 (Google / Gemini Pro):** 1차 교차검증/논리 점검
5. **5번 뇌 (로컬 엔진):** 데이터 수집/배치/자동화 전담
6. **6번 뇌 (Anthropic / Claude Sonnet 4.5):** 코드리뷰/문서검토 (2차 검증)
7. **7번 뇌 (Anthropic / Claude Opus 4.5):** 최종 심화검증/고난도 추론

> 삭제 반영: 기존 o1, Antigravity 3 Pro, o3-mini(권한 차단), GPT 5.2 Codex 항목 제거

## 역할 분업 (고정)

### 수집 파이프라인
- **1차 수집:** 5번(로컬) + 1,2번(Flash) — 대량 크롤링, 증분, 재시도

### 코딩/구현 파이프라인
- **구현:** 3번 똑뇌 (메인)
- **1차 리뷰:** 6번 Sonnet (안정형, 빠름)
- **최종 리뷰:** 7번 Opus (복잡한 케이스만)

### 전략/검증 파이프라인
- **전략 설계:** 3번 똑뇌
- **1차 교차검증:** 4번 Gemini Pro (비용 효율)
- **2차 심화검증:** 6번 Sonnet
- **최종 승인:** 7번 Opus (고난도 추론 필요 시)

### 아이디어/브레인스토밍 파이프라인
- **참여:** 7번(Opus) + 6번(Sonnet) + 4번(Gemini Pro)
- **종합/판단:** 3번 똑뇌
- **최종 승인:** 주인님

```
[브레인스토밍 흐름]
주인님 요구사항 → 7,6,4번 동시 아이디어 수집 → 3번 종합 → 주인님 승인
```

### 코딩/구현 의사결정 흐름
```
3번(구현) → 6번(1차리뷰) → 4번(교차검증) → 7번(최종승인) → 배포
```

## 운영 원칙
1. 반복·대량 작업은 반드시 수집 담당으로 분리한다.
2. 똑뇌는 수집 직접 수행보다 "검증/판단"에 우선 자원을 쓴다.
3. 비용/속도/안정성 기준으로 분업이 깨지면 즉시 복구한다.
4. 주인님 지시가 있으면 언제든 우선순위를 재배치한다.
5. 추가 과금 금지: 기존 구독(OAuth/CLI) 범위 외 신규 결제·유료 API 호출은 사전 승인 없이는 금지.
