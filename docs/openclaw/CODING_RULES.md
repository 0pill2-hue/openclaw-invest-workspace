# CODING_RULES.md

Last updated: 2026-02-18 06:47 KST
Purpose: 게이트 기반 개발에서 안정성/재현성/추적성을 유지하기 위한 코딩 규칙

## 1) 설계 원칙
- 기능 코드 / 단계 실행 코드 / 게이트 코드 / 운영 오케스트레이션 코드를 분리한다.
- 단계 실패 시 다음 단계 실행 금지(게이트 우선).
- 매직넘버 금지: 임계치/패널티는 상수 또는 설정으로 분리.

## 2) 구현 규칙
- Python: PEP8 + type hint(핵심 함수 필수)
- 함수는 단일 책임 유지(입력검증/계산/저장 분리)
- 예외는 삼키지 말고, 원인 포함 로그를 남긴다.
- 파일 경로는 하드코딩 최소화(기준 루트 상수화)

## 3) 산출물/결과 규칙
- 결과 파일은 `grade` 필드 필수 (`DRAFT|VALIDATED|PRODUCTION`)
- `DRAFT` 결과는 `TEST ONLY` 워터마크 필수
- 결과 경로 분리 원칙 준수 (`test/validated/prod`)

## 4) 검증 규칙 (완료조건)
- 코드 변경 직후 즉시 실행 검증(스모크 최소 1회)
- 게이트 관련 변경은 대응 gate check 통과 로그를 proof로 남긴다.
- 미검증 상태에서 완료 보고 금지

## 5) 변경 관리
- 시스템/전략 영향 변경 시 문서 동기화 필수
  - `docs/openclaw/*`
  - `invest/strategy/RULEBOOK_*`
  - `reports/stage_updates/stage*.md`
- 커밋 단위는 기능/수정 목적 기준으로 작게 유지
