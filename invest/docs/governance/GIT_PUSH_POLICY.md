# GIT_PUSH_POLICY.md

Last updated: 2026-02-19 17:39 KST
Purpose: 커밋 후 push 지연/누락 방지

## 기본 원칙
1. `main` 작업은 커밋 완료 후 **즉시 push**가 기본이다.
2. 동일 작업에서 커밋이 2개 이상 쌓이면, 마지막 커밋 직후 **즉시 push**한다.
3. 예외는 아래 2개만 허용:
   - 민감정보 점검 중(누출 검증 미완료)
   - 대용량 산출물 정리 중(경로/ignore 정합성 미완료)
4. 예외 상태는 반드시 작업 로그에 이유/예정 push 시각을 남긴다.

## 운영 체크리스트 (커밋 직후)
- `git status --short` 확인
- `git push origin main` 실행
- 실패 시 즉시 원인 기록(권한/네트워크/충돌)

## 금지
- "나중에 한 번에 push" 관행
- 다수 커밋 누적 후 무기한 로컬 보관
