# Value Rule v2 (DRAFT)

> 상태: DRAFT (교차리뷰 전)
> 목적: 내지표/이웃지표를 공통 기준으로 재산출하기 위한 임시 합의 문서

## 1) 공통 산출 원칙
- 동일 윈도우: 20일(단기), 60일(중기)
- 동일 정규화: 퍼센타일 기반 0~100 (p5~p95 clip)
- 동일 비교 단위: 비교 가능한 항목만 `diff = my - neighbor` 계산
- 비교 불가 항목: 별도 섹션으로 분리(우열 판단 금지)
- 타임스탬프: 저장 UTC, 보고 KST 병기

## 2) 공통 코어 피처(비교 가능)
1. trend_slope_ma_cross
   - 정의: ((MA20 - MA60) / MA60) * 100
2. roc_20d
   - 정의: 20일 수익률(%)
3. volume_ratio_20d
   - 정의: 현재 거래량 / 20일 평균 거래량
4. volatility_20d_inv
   - 정의: 20일 변동성 점수를 역변환(낮을수록 높은 점수)

## 3) 이웃 전용 정성 피처(비교 불가, 별도 관리)
- interest_axis (관심축)
- directionality (방향성)
- consistency (일관성)
- reliability (신뢰도)

규칙:
- 내지표에 대응 항목이 없으면 `my_score = null`
- `diff` 계산하지 않음
- 결과 해석은 절대 우열표현 금지

## 4) 필터링 규칙 (블로그/텍스트)
- 모집/판촉/광고성 텍스트는 기본 하드 필터(제외)
- 예외 통과 조건: 실적/수치/근거 링크가 명시된 경우만 보조 반영

## 5) 리스크/방어 규칙
- DC-01: 결측 3회 연속 시 입력 차단, 3회 정상 수신 시 복구
- AL-01: 1시간 동일방향 신호 과밀(기준 x3) 시 신규 진입 1시간 홀드
- RP-01: 약속 보고 +15분 SLA 초과 시 즉시 알림

## 6) 산출물 거버넌스
- 등급: DRAFT | VALIDATED | PRODUCTION
- DRAFT 결과물은 `TEST ONLY` 필수
- 저장 경로 분리:
  - test/: DRAFT
  - validated/: VALIDATED
  - prod/: PRODUCTION

## 7) 리뷰 게이트 (필수)
- 설계 리뷰 1회 이상
- 구현 리뷰 1회 이상
- 결과 리뷰 1회 이상
- 미리뷰 결과는 DRAFT로만 취급

## 8) 다음 단계
1. 이 문서 기준으로 피처 재산출
2. 교차리뷰(설계/구현/결과) 수행
3. 리뷰 반영 후 VALIDATED 문서로 승격
